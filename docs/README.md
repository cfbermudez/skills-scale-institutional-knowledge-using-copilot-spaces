# OctoAcme Project Management Docs

This folder contains the process documentation for OctoAcme's project management practices. These docs serve as the single source of truth for how OctoAcme plans, executes, and delivers product initiatives. Use them to onboard new team members, align on workflows, and maintain consistency across projects.

## Overview

OctoAcme's project management approach covers the full project lifecycle — from validating a business need through retrospective — with clear roles, artifacts, and communication rhythms at each stage.

**Core principles:**
- Customer-first delivery and measurable outcomes
- Iterative, incremental planning and release
- Clear ownership: every project has a named Project Manager (PM) and Product Lead
- Standard artifacts and checkpoints (charter, backlog, risk register, retrospective notes)
- Consistent risk management and communication cadence

**Key roles:** Project Manager, Product Manager (PdM), Developers, QA/Testing, Stakeholders.
